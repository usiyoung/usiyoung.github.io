Redux 란?

리덕스는 javascript application들의 상태(state)를 관리하는 방법이다. 리덕스는 리액트에 자주 사용돼 리액트에 의존적인 라이브러리라 생각할 수 있지만, 리덕스와 리액트는 별도의 것이다. 뷰나 바닐라 자바스크립트에도 사용할 수 있다.

Redux 시작하기

  1. yarn add redux : 리덕스 설치
  2. import {createStore} from 'redux' : createStore 임포트 하기

store은 데이터(state)를 넣는 곳이다. 다시 한 번 정리하면, 리덕스는 state를 관리할 수 있게 도와주는 역할을 하고, 데이터들은 store에 들어간다. 리덕스 = state 관리자 / store = 저장소

Store and Reducer

  • store 생성하기

    store을 생성하게 되면 reducer는 함수가 되어야 한다고 에러가 뜬다.

    error

    다시 돌아가 createStore 메서드를 확인해보면 createStore에 reducer를 넣어줘야 한다고 말하고 있다.위 에러에서 reducer는 함수형태라고 했으니 함수형태의 reducer를 전달해주면 된다.

    reducer를 전달해줘

    여기서 정리 한번, store을 생성하고 함수형태의 reducer를 전달해줘야 한다. 그럼 이 reducer의 역할은 무엇일까?

  • reducer 사용하기
    reducer안에다가 리턴값으로 텍스트를 남겨둔 후 store의 데이터를 불러오는 getState메서드를 통해 콘솔을 출력해보면 hello가 출력된다. 즉, reducer는 데이터를 수정가능하게 해준다.

    데이터를 수정하려면 reducer를 통해 관리하면 된다. 여러 데이터를 한 곳에서 관리할 수 있어 편리함도 준다.

  • state 이니셜라이징

    reducer에 state를 인자값으로 넣어줄 때 값이 없을 경우 undefined가 나오게 된다.

    이니셜라이징 할 수도 있다. 이니셜라이징을 해주면 인자의 값이 없을 때를 방지해줄 수 있다.

Action

action은 위의 reducer와 소통할 수 있게 도와준다. 위의 코드에서 state 값을 변경하기 위해 action을 사용한다. action을 사용해주기 위해 reducer의 두번째 인자에 넣어둔다.

두번째 인자에 넣어준 뒤 action에 값을 넘기려면 dispatch메서드를 사용한다. dispatch 메서드 안에는 객체만 들어갈 수 있고, key를 type으로 넣어줘야 한다.

dispatch메서드로 값을 던져주면 action이 받아 값을 변경할 수 있다. 이런식으로 dispatch로 던져주면 action이 받고 그걸 토대로 데이터를 수정할 수 있다.

이런식으로 코드를 짤 수도 있다. 플러스 버튼을 누르면 값이 1씩 증가하고 마이너스 버튼을 누르면 값이 1씩 감소한다.

Subscribe

subscribe는 state값이 변경 됐을 때 호출된다. createStore의 메서드 subscribe를 활용해 action처럼 함수를 전달해주면 된다.

간단 정리

  • redux는 상태를 관리하는 라이브러리이며,
  • createStore을 통해 저장소를 만들 수 있고,
  • reducer로 저장소안의 데이터를 수정할 수 있으며,
  • dispatch 메서드를 통해 action에 던져주면 reducer와 소통할 수 있다.
  • 상태가 바뀔 때 알려주는 subscribe도 있다.
